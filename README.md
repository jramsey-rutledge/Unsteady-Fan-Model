# Unsteady-Fan-Model
This code solves the unsteady advection-diffusion equations for a room with a fan. 

**Author:**
- Jenna Ramsey-Rutledge

## Background
This project models two-dimensional heat transport in a room with internal air circulation generated by a ceiling fan. The governing physics include both thermal diffusion and advection caused by the moving air. The solver was developed to study transient heat transport, visualize mixing behavior, and verify numerical accuracy using manufactured solutions.

The code is written in C++ using a finite-volume discretization on a structured Cartesian mesh. Sparse linear systems are assembled and solved implicitly at each timestep. MATLAB post-processing scripts are included for visualization, error analysis, and animation generation.

## Physics and Mathematical Model

### Governing Equations
The temperature $T(x,y)$ satisfies the steady advection-diffusion equation written in integral form, 

$$\int_{CS} (\rho c_{p} \vec{u} T) \cdot \vec{n} dA = \int_{CS} k (\nabla T \cdot \vec{n}) dA + \int_{CV} f dV,$$

where $\rho$, $c_p$, and $k$ are the density, specific heat, and theraml conductivity of air, which we assume are constants. The velocity field is defined $\vec{u}=(u,v)$. The source term $f$ is included both for verification using the Method of Manufactured Solutions (MMS) and for modeling internal heat generation such as a heater.
The fan is modeled using an incompressible velocity field described as 

$$
\begin{align}
u(x,y) & = -\frac{A}{b_yL_y}\sin{\left(\frac{2\pi x}{b_xL_x}\right)}\cos{\left(\frac{2\pi y}{b_yL_y}\right)} m/s \\
v(x,y) & = \frac{A}{b_xL_x}\cos{\left(\frac{2\pi x}{b_xL_x}\right)}\sin{\left(\frac{2\pi y}{b_yL_y}\right)} m/s,
\end{align}
$$

where $A$ (units m/s) controls the velocity amplitude, and $b_x$ and $b_y$ are perscribed dimensionless constants that determine the spatial structure of the flow. This velocity field is divergence-free and produces recirculating flow patterns similar to a ceiling fan.

### Initial and Boundary Conditions
This solver supports both Dirichlet and Neumann boundary conditions through a generalized Robin boundary-condition framework,

$$aT + b\frac{\partial T}{\partial n} = g.$$

For the room simulations, all of the walls are insulated using homogeneous Neumann conditions. However, the south wall contains an open window held to a fixed lower temperature and the east wall contains a door fixed at a higher temperature. 
The initial condition may be specified uniformly or through a user defined distribution. The manufactured solution requires analytical initial and boundary conditions consistent with the exact solution. 

## Numerical Methods 
The governing equations are discretized using the finite-volume method on a structured Cartesian grid with cell-centered unknowns. The spatial discretization utilizes central differencing. The diffusion terms are handled with second-order central differences and the advection terms use a centered finite-volume flux formulation. The face fluxes are computed via linear interpolation of the velocity values on cell faces. 

As for the temporal discretization, an implicit first-order backward Euler method is used,

$$ \frac{\partial T}{\partial t} \approx \frac{T^{n+1} - T^{n} }{\delta t}. $$

This produces a sparse linear system at each timestep,

$$ AT^{n+1} = b. $$

The sparse matrix system is assembled using Eigen sparse matrices and solved using LU factorization. 

## Verification

The solver was verified using the Method of Manufactured Solutions. The test solution

$$f = \cos(x)\cos(y)\cos(\omega t)$$

is substituted into the governing equations to derive a compatible source term $f(x,y,t)$. 

To verify spatial convergence, grid refinement studies were ran for grid sizes of 16x16 to 128x128 and marched in time until a steady-state solution was reached. A test script collected this data and produced a second-order error plot. A similar process was done to produce a plot showing first-order temporal accuracy. 

## Quick Start

### Prerequisites

- C++17-compatible compiler
- GNU Make
- Eigen 5.0.0+

### Building and Running

```bash
# Clone the repository
git clone <repo-url>
cd Unsteady-Fan-Model

# Compile the project
make build

# Run the program
./build/solver.exe

# Remove compiled artifacts
make clean

# Build and run in one step
make all
```

### Adjusting the Simulation

Edit the .inp files in `input/` to change the spatial domain, temporal domain, fan parameters, or physical properties before building
